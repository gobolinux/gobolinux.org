GoboLinux 017.01 - Release Notes

Introduction

We are pleased to present release 017.01 of GoboLinux, the Linux distribution featuring a rethought file system structure. GoboLinux was created out of a desire to try new approaches in the Linux distribution design space. GoboLinux 017 continues this journey, with a focus on the exploration of novel ideas aiming to make the system simpler yet functional..

GoboLinux 017.01 is an eagerly awaited bug-fix update that addresses several critical issues since our initial 017 ISO release.

Along with this release, Hisham Muhammad, founder and steward of GoboLinux for the past 25 years, is officially stepping down! The project now continues under the stewardship of Philip Pok (@nuc1eon).

Availability

GoboLinux is available as an ISO download which you can download from this site.

Overview of this release

Main changes:

  • Resolved the following documented Known issues and fixes since our 017 ISO release:
    • Include UnionFS-Fuse to workaround limitations with the sandbox in the live environment
    • Loopback network interface needs to be brought up at boot
    • Update the search URL used by the Scripts package
    • Ncurses had been replaced by NcursesW, but some packages' metadata still held references to the former
    • Update expired SSL certificates (thanks to Jean-Michel T.Dydak @yann-kaelig)
    • Fix wrong versioning of SQLite
  • Fixed a missing symlink (/usr/libexec → /System/Index/libexec)
  • Fixed some startup Tasks that where not marked as executables (Avahi, DHCPNetworkInterface)
  • Installer: Fix ClockMode (GMT/localtime) selection and default to GMT
  • Installer: Add support for eMMC storage devices (thanks to @TheBitStick)
  • Freshen has been ported over to Python 3 (thanks to Sage I. Hendricks @sage-etcher) and been therefore re-added to Gobolinux (experimental state)
  • Updated some core packages, bumped kernel to v6.12.16 (special thanks to Samuel Dionne-Riel @samueldr for the updated GoboHide patch)
  • Updated major GoboLinux tools with lots of fixes (details not listed here)
  • Switched kernel, initramfs, firmware and squashfs to zstd compression
  • Updated Firefox default profile
  • Switch on numlock during boot

Known outstanding issues

  • In a VM environment, installing GoboLinux' bootloader onto a GPT/UEFI setup fails. Please set up an MBR partition table. On a bare metal system GPT partitions work fine in our testing (if in doubt, configure your bootloader manually).
  • Copy & Paste between VM and and host system does not work
  • Our documentation has accumultaed some cruft over the years and is in the process of being updated. Contributions welcome!:-)

Any help solving these issues for our next release is appreciated!

Expectations

As an experimental distribution, you might run into some bugs along the way. We encourage you to tackle these challenges on your own or with the support of our friendly community. While we can't promise everything will be handed to you, your discoveries and solutions are invaluable to us—please share them!

GoboLinux thrives on the passion and contributions of its users, and together, we can make it even better.

Outlook

Stay tuned for another point release this year, this time we will focus on updating our base packages - and hopefully squash more bugs, too!
